# Repo AGENTS.md (PodcastTranscriptor)

## Scope
These instructions apply to this repository root: `.`.

## Purpose
This file is for project-level guidance and preferences specific to this repository.

## Organization
Keep work highly organized. Store project artifacts in clear subfolders under this repo, and keep operations isolated from personal system state.

## Environment
Use the Conda environment named `pds_env` for this project workflow.

## Investigation Logging
When an investigation is done, append an extremely concise summary of what was learned to `dev_log.md` at the repo root.

## Dev Log Maintenance
Keep the dev log compact. If a line of investigation is later superseded, remove detailed dead-end notes and keep only a short note: what was tried, why, and why it was replaced.

## Dev Log Format
Every dev log update must use a proper Markdown heading in the form `## YYYY-MM-DD HH:MM:SS TZ`.
Do not append loose top-level bullet points without a heading.
